ECMAScriptについて

1.ECMAScriptについて


ECMAScriptはJavaScriptを標準化させた仕様の言語。

MDNは一通り目を通しておきましょう！
https://developer.mozilla.org/ja/docs/Web/JavaScript

ECMAScriptについて
https://developer.mozilla.org/ja/docs/Web/JavaScript/Language_Resources

1-1.ECMAScriptの誕生について


JavaScriptが誕生したのは1995年。
Netscape社がWebで実行出来るスクリプト言語という事で開発された。
その後、Internet Explorerに搭載されて浸透していったが、
当時のブラウザは今と違って、独自機能や拡張が多くて互換性がとても低い状態だった。
そこで、Ecma International という標準化団体が中心となり、標準化をしていき、
1997年にECMA-262(ES1)が誕生。

ES2015とかES6とかES7とか色々あるけど何が最新なのか？
→ES2018(ES9)が最新です。ES5とか何！ES2015ならES5じゃないの！って感じですが
　次のスライドで歴史について説明します。

1-2.ECMAScriptの歴史について


1-3.仕様策定プロセスの変化


【ES2015以前】
-> 全ての仕様の合意が取れてから一括リリース
【ES2016以降】
-> 合意がとれた仕様から順次リリースし、 毎年リリースする策定プロセスに変化。

それにより、ES6,ES7,ES8という名称から、年表記に切り替えられ、
ES2015、ES2016、ES2017、ES2018 という呼び方に変わった。

2.ECMAScriptの仕様の流れ


ECMAScriptは毎年6月頃にリリースされる。
今回はさらっと説明するだけなので詳しくは下記公式をご覧ください。
【ES2015】 http://www.ecma-international.org/ecma-262/6.0/
【ES2016】 https://www.ecma-international.org/ecma-262/7.0/
【ES2017】 https://www.ecma-international.org/ecma-262/8.0/
【ES2018】 https://www.ecma-international.org/ecma-262/9.0/

2-1.ES2015(ES6)で追加された機能


ES2015(ES6)は、ECMAScriptの基本バージョンとなる為、沢山の機能がある。

・アロー関数 (Arrows functions)
・class構文 (classes)
・let/const宣言 (let and const)
・Promise
・モジュール構文 (modules)
・関数のデフォルト引数 (default + rest + spread)
・ジェネレータ
・配列展開 (Array Spread)
・テンプレート文字列 (Template Strings)
・分割代入 (destructuring)

その他にも
Map/Set/WeakMap/WeakSet、math/number/string + array + object APIs等.....

2-2.ES2016(ES7)で追加された機能


ES2016の変更はとても小規模で以下の2機能のみ

・Array.prototype.includes
-> 配列に任意の要素が含まれているか含まれていないかチェックするメソッド
・Exponentiation Operator
-> 指数演算子の機能を持っていて数学向けアプリを実装する際に最適ですが忘れて下さい

2-3.ES2017で追加された機能


ここからはES8ではなくES2017と年表記で説明します。※一応ES8とも言われている。
ES2017もES2016同様に機能追加は少ないのですが便利な機能がいくつか追加。

一番重要なのが、「Async functions (Async/Await)」
こちらは非同期関数で、Promiseと組み合わせる事でES2016以前で問題になっていた非同期処理が改善されている。

その他に
・String padding
・Object.values
・Object.entries
・getOwnPropertyDescriptors
・Trailing commas

2-4.ES2018で追加された機能


ES2018は以下の5つが最近リリースされたばかり。
・オブジェクトの Rest/Spread プロパティ（Object Rest/Spread Properties）
・Promise.prototype.finally()
・テンプレートリテラルの改修（Template Literal Revision）
・正規表現（RegExp）関係
・Asynchronous Iterators
見直しや正規表現周りに強くなり、Promiseに機能追加等ザックリそんな感じです

ハンズオン

3.JavaScriptハンズオン


ECMAScriptの前にJavaScriptなのか、ECMAScriptなのか混在していると思います。
なので、まずはJavaScriptの基礎から抑えて行きましょう！
抑えるべき点としては以下の4点です。(基本構文(if, for, while)は省きます)

(1) 変数
(2) 配列
(3) データ型 (プリミティブ型、オブジェクト型)
(4) 関数とスコープについて

3-1-1.変数について


従来のJavaScriptでの変数宣言は「var」のみ。
数値、文字列、booleanどれもvarを使います。
この１つの宣言方法でも色々ありますのでご紹介します。

var nickname = "うい氏";

3-1-2.変数について


◼varの使い方
基本的な使い方として以下の形になります。
var 変数名 = 値;

◼複数での宣言が出来る
,(カンマ)を使用する事により複数で宣言する事も出来ます。

◼var有りと無しで挙動が変わる
varと変数名のみ宣言した場合は、値は undefined になるが宣言した形になります。
varを使用しないで変数名のみ宣言した場合は、エラーになります。

3-1-3.変数について


～ハンズオンタイム～


(1) varの宣言と代入
(2) varの初期化を含めた宣言&再代入
(3) 複数変数の宣言

3-2-1.データ型について


JavaScriptのデータ型には大きく分けると
プリミティブ型 (primitive) とオブジェクト型 (object) の２つが存在する。

// 数値 var val1 = 123; // number // 文字列 var val2 = "テキストで～す"; // string // ブール var val3 = true; // boolean // オブジェクト var val4 = { key: "value" } // object

3-2-2.データ型について


【プリミティブ型 (primitive)】
・数 (number)
・文字列 (string)
・ブール型 (boolean)
・null (ES以降は null という型になる) ※typeofで確認するとobjectと返却される為、注意
・undefined

【オブジェクト型 (object)】
・オブジェクト(object)
・関数(function)

これらのデータ型は typeof演算子 で判別する事が出来ます。

3-2-3.データ型について


オブジェクト型 (object) について

【Wikipedia の場合】
関連するデータを束ね、代入、演算、手続き（関数やメソッドなど）を
介した受け渡しといった操作の対象にでき、またメッセージの受け手になれる実体

【サイ本 (O’Reilly) の場合】
名前の付けられた値を集めたもの

var obj = { key1 : "value1", // keyが名前(キー) key1 : "value2" // valueが値(バリュー) };

3-2-4.データ型について


～ハンズオンタイム～


(1) プリミティブ型の宣言とtypeof演算子確認
(2) 型変換(キャスト)をして型の確認&再代入
(3) オブジェクト型の宣言と確認
(4) オブジェクトにデータの追加と削除

3-3-1.配列について


変数はオブジェクトを含めて、１つの値しか格納出来ないが、
配列は複数の値を格納する為に使用される。
配列にはindexが割り振られていて、indexによりアクセスする事も出来る。

// ID一覧 var idList = [12, 34, 78, 90]; // ユーザー一覧 var userList = ["taro", "hanako", "bangu", "sawada"];

3-3-2.配列について


【配列に格納出来る値】
配列に格納するデータ型に制約は無く、文字列・数値・ブール・オブジェクト・関数いずれも保持する事が可能。

【配列の定義について】
配列の定義方法は「配列リテラル式」と「new式」の２つ。
new式の場合、Arrayインスタンスが出来上がる為、コンストラクタが呼び出されますが実際出来上がる物は同じ。
※new式は少し古いやり型で今はコードの可読性を良くする為に配列リテラル式(直接定義)が主流であり好まれます。

3-3-3.配列について


【配列の種類について】
avaScriptには通常の配列以外に２つの配列が存在する。

【多次元配列】
多次元配列は配列の中に配列を含める事により多次元にアクセスが出来る。

var list = [ ["りんご", "いちご"], // 一つ目の配列は赤い食べ物 ["キウイ", "メロン"] // 二つ目の配列は緑の食べ物 ]; console.log(list[0][0]); // りんご console.log(list[1][0]); // キウイ

3-3-4.配列について


【連想配列 (Dictionary)】
JavaScriptの連想配列は少しややこしくなりますが、オブジェクトと同じ。 indexキーが文字列に変わるだけです。

実際のプロジェクトでは配列とオブジェクトの掛け合わせが主流。

var obj = { key1 : "value1", // key が名前(キー) key2 : "value2" // value が値(バリュー) };

3-3-5.配列について


～ハンズオンタイム～


(1) 配列の宣言と定義
(2) 配列の長さを取得
(3) for文で回す
(4) オブジェクトを組み合わせた多次元配列にして、for..in文で回す

3-4-1.関数とスコープについて


JavaScriptは関数型プログラミング言語とも呼ばれている。

function myFunc(name) { var message = "I'm " + name; console.log(message); }

3-4-2.関数とスコープについて


【関数とは (MDNより)】
関数は JavaScript の基本的な構成要素のひとつ。
また関数は、JavaScript の手続き ― つまり、タスクや値計算を実行する文の集まり。

// 名前付き関数 function 関数名(引数) { 処理内容; return 戻り値; } // 無名関数 var 変数名 = function(引数) { 処理内容; return 戻り値; }
