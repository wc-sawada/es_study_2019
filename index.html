ECMAScriptについて

1.ECMAScriptについて


ECMAScriptはJavaScriptを標準化させた仕様の言語。

MDNは一通り目を通しておきましょう！
https://developer.mozilla.org/ja/docs/Web/JavaScript

ECMAScriptについて
https://developer.mozilla.org/ja/docs/Web/JavaScript/Language_Resources

1-1.ECMAScriptの誕生について


JavaScriptが誕生したのは1995年。

Netscape社がWebで実行出来るスクリプト言語という事で開発された。
その後、Internet Explorerに搭載されて浸透していったが、
当時のブラウザは今と違って、独自機能や拡張が多くて互換性がとても低い状態だった。

そこで、Ecma Internationalという標準化団体が中心となり、標準化をしていき、
1997年にECMA-262(ES1)が誕生。

ES2015とかES6とかES7とか色々あるけど何が最新なのか？
→ES2018(ES9)が最新です。(今年ES2019リリース予定)
　ES5とか何！ES2015ならES5じゃないの！って感じですが
　次のスライドで歴史について説明します。

1-2.ECMAScriptの歴史について


1-3.仕様策定プロセスの変化


【ES2015以前】
-> 全ての仕様の合意が取れてから一括リリース

【ES2016以降】
-> 合意がとれた仕様から順次リリースし、 毎年リリースする策定プロセスに変化。

それにより、ES6,ES7,ES8という名称から、年表記に切り替えられ、
ES2015、ES2016、ES2017、ES2018 という呼び方に変わった。

2.ECMAScriptの仕様の流れ


ECMAScriptは毎年6月頃にリリースされる。

今回はさらっと説明するだけなので詳しくは下記公式をご覧ください。
【ES2015】 http://www.ecma-international.org/ecma-262/6.0/
【ES2016】 https://www.ecma-international.org/ecma-262/7.0/
【ES2017】 https://www.ecma-international.org/ecma-262/8.0/
【ES2018】 https://www.ecma-international.org/ecma-262/9.0/

2-1.ES2015(ES6)で追加された機能


ES2015(ES6)は、ECMAScriptの基本バージョンとなる為、沢山の機能がある。

・アロー関数 (Arrows functions)
・class構文 (classes)
・let/const宣言 (let and const)
・Promise
・モジュール構文 (modules)
・関数のデフォルト引数 (default + rest + spread)
・ジェネレータ
・配列展開 (Array Spread)
・テンプレート文字列 (Template Strings)
・分割代入 (destructuring)

その他にも
Map/Set/WeakMap/WeakSet、math/number/string + array + object APIs等.....

2-2.ES2016(ES7)で追加された機能


ES2016の変更はとても小規模で以下の2機能のみ

・Array.prototype.includes
-> 配列に任意の要素が含まれているか含まれていないかチェックするメソッド
・Exponentiation Operator
-> 指数演算子の機能を持っていて数学向けアプリを実装する際に最適ですが忘れて下さい

2-3.ES2017で追加された機能


ここからはES8ではなくES2017と年表記で説明します。※一応ES8とも言われている。
ES2017もES2016同様に機能追加は少ないのですが便利な機能がいくつか追加。

一番重要なのが、「Async functions (Async/Await)」
こちらは非同期関数で、Promiseと組み合わせる事でES2016以前で問題になっていた非同期処理が改善されている。

その他に
・String padding
・Object.values
・Object.entries
・getOwnPropertyDescriptors
・Trailing commas

2-4.ES2018で追加された機能


ES2018は以下の5つが最近リリースされたばかり。

・オブジェクトの Rest/Spread プロパティ（Object Rest/Spread Properties）
・Promise.prototype.finally()
・テンプレートリテラルの改修（Template Literal Revision）
・正規表現（RegExp）関係
・Asynchronous Iterators

見直しや正規表現周りに強くなり、Promiseに機能追加等ザックリそんな感じ。

3-3-1.配列について


変数はオブジェクトを含めて、１つの値しか格納出来ないが、
配列は複数の値を格納する為に使用される。

3-3-2.配列について


【配列に格納出来る値】
配列に格納するデータ型に制約は無く、文字列・数値・ブール・オブジェクト・関数いずれも保持する事が可能。

【配列の定義について】
配列の定義方法は「配列リテラル式」と「new式」の２つ。
new式の場合、Arrayインスタンスが出来上がる為、コンストラクタが呼び出されますが実際出来上がる物は同じ。
※new式は少し古いやり型で今はコードの可読性を良くする為に配列リテラル式(直接定義)が主流であり好まれます。

【配列の種類について】

【多次元配列】
多次元配列は配列の中に配列を含める事により多次元にアクセスが出来る。

【連想配列 (Dictionary)】
JavaScriptの連想配列は少しややこしくなりますが、オブジェクトと同じ。

【グローバルスコープとは】
関数外のスコープの事。関数外で宣言された変数や関数はグローバルスコープになる。

【ローカルスコープとは】
関数内のスコープの事。関数内で定義された変数や関数はローカルスコープになる。
